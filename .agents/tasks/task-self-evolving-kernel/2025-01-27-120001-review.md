All five issues from the v1 review have been addressed. The immutability guard now normalizes paths via `os.path.normpath()` and detects directory traversal by resolving against `kernel_dir`. The runner documents its "always take first transition" behavior as intentional Mode 1 scaffolding. BOOT.md clarifies state ownership per execution mode. **Verdict**: APPROVED

## High-level view

The path normalization fix applies `os.path.normpath()` to both the `IMMUTABLE_FILES` frozenset at module load and every incoming path field in `validate_change`. A second layer uses `Path.resolve()` to detect traversal escaping `kernel_dir`, catching absolute paths (Python's `/` operator on Path treats absolute RHS as the new root) and symlink-following attacks. The dual-layer approach — normpath for string comparison, resolve for containment — provides defense in depth.

The rollback fix stores node state and file content at apply time (before the destructive operation), then logs the enriched change dict to history.jsonl. One edge case remains: newly created prompt files have no `original_content` to restore, so rollback leaves them behind rather than deleting.

The dual-mode documentation (runner.py module docstring, inline SCAFFOLDING comment, BOOT.md state-ownership block) eliminates the ambiguity about who owns `state.yaml` in each execution mode.

## Rollback fidelity gap

The `apply_change` path for `modify_prompt` only saves `original_content` when `prompt_path.exists()`:

```python
if prompt_path.exists():
    change["details"]["original_content"] = prompt_path.read_text(encoding="utf-8")
```

If the file is newly created (e.g., adding a new prompt via evolution), no `original_content` is stored. The rollback guard `if prompt_file and original_content:` then evaluates to false (empty string is falsy), and the file remains on disk. A clean rollback should detect the absence of `original_content` as a signal to `os.unlink()` the created file rather than silently leaving it. This is low-risk — evolution typically modifies existing prompts — but would matter if the kernel starts creating new prompt files as part of its evolution cycle.
